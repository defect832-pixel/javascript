System Output

console.log()

This is your primary diagnostic tool. It serializes data and prints it to the hidden DevTools console. Think of it as a system report that only you (the engineer) can see.

main.js
1 // Send message to system logs
2 console.log("System Ready");
3 console.log("User ID: 402");
console.error()

Used for critical failures. Unlike a standard log, this outputs a Stack Trace, showing exactly which file and line caused the crash. Essential for debugging complex logic.

error_handler.js
1 if (disconnected) {
2   console.error("Connection Lost!");
3 }

User Interface Interaction

window.alert()

Triggers a native browser modal.
Note: This is a "blocking" action. The code stops completely until the user clicks OK.

1window.alert("Access Denied");
document.write()

Injects raw text directly into the HTML stream. Dangerous if used after page load, as it will overwrite the entire document.

1document.write("<h1>Hi</h1>");
window.close()

Attempts to close the tab. Browsers limit this for security; it usually only works if the script opened the window itself.

1window.close();
